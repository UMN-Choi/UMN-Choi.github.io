Reproducibility

We commit to the following for every released repo:

  • Pinned dependencies · requirements.txt or environment.yml
  • Single-command training entrypoint · scripts/train.sh
  • Single-command evaluation · scripts/eval.sh
  • Pre-trained checkpoints when license permits
  • Dataset DOI or download script
  • Hardware & wall-clock from the paper (partial)
  • Hyperparameter search logs (partial)

Issues for missing artifacts: file at github.com/UMN-Choi-Lab/<repo>/issues — tag with repro.
